# decisions/

An ADR log for decisions with lasting consequences — the ones worth remembering the *why* behind, not just the *what*.

The assistant should:

- Add a new numbered entry to `_index.md` when a hard-to-reverse or cross-project decision is made (with you, not guessed).
- Before closing a large change, check the diff against relevant ADRs here — flag it if the change contradicts or should update one, rather than silently drifting from a recorded decision.
- Prefer updating an ADR's "Consequences" over creating a near-duplicate entry when a decision evolves.
