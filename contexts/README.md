# contexts/

Switchable "modes" you can ask the assistant to load for a specific kind of
work — e.g. `dev` for active feature building, `review` for PR/code review.
Each file sets the focus, behavior, and priorities for that mode.

Ask the assistant to "load the dev context" or "switch to review mode" to
activate one. Edit the existing files to match how you like to work, and add
new ones for other modes you use often (e.g. `debug.md`, `writing.md`).
