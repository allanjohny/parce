# Harness setup

One canonical file — `ASSISTANT.md` — symlinked to whatever each tool expects. `setup.sh` creates all of these automatically; this doc explains what goes where and per-tool tips.

## The symlink map

```
ASSISTANT.md                        ← canonical, generated by setup.sh
├── CLAUDE.md                       ← Claude Code
├── AGENTS.md                       ← Cursor, OpenAI Codex, Zed, Jules, most agent CLIs
└── .github/copilot-instructions.md ← VS Code GitHub Copilot
```

Edit `ASSISTANT.md` (or re-run `./setup.sh` to regenerate from the template) and every tool picks up the change.

> **Windows note:** symlinks require Developer Mode or an elevated shell. If symlinks aren't available, copy the file instead — you lose single-source-of-truth but everything still works.

## Claude Code

Works out of the box: opening this repo loads `CLAUDE.md`.

Optional power-ups (in `~/.claude/settings.json` or via `claude config`):

- **Auto-memory**: point `autoMemoryDirectory` at this repo's `memory/` so Claude Code's built-in memory feature writes here.
- **Skills**: symlink `skills/brain` into `~/.claude/skills/` to invoke the brain rebuild as a skill from any project:
  ```bash
  ln -sfn "$(pwd)/skills/brain" ~/.claude/skills/brain
  ```

## Cursor

Cursor reads `AGENTS.md` from the repo root (and `.cursor/rules/` for granular rules). The symlink covers the basics. If you want the assistant identity available in *every* project, add the contents to Cursor's global "Rules for AI" setting.

## VS Code (GitHub Copilot)

Copilot reads `.github/copilot-instructions.md`. Enable it in settings if needed:
`"github.copilot.chat.codeGeneration.useInstructionFiles": true`.

## OpenAI Codex / Zed / other agent CLIs

Most modern agents converged on `AGENTS.md` at the repo root — already covered by the symlink.

## Anything else

Point the tool's "custom instructions" or "system prompt" feature at `ASSISTANT.md`. It's plain Markdown; every tool can read it.

## Using your assistant across projects

Two patterns:

1. **Hub repo (recommended to start):** this repo is the assistant's home. Open your AI tool here for personal/organizational tasks; open other repos normally for coding. Memories about other projects still live here.
2. **Global install:** symlink `ASSISTANT.md` into your tool's global instructions location (e.g. `~/.claude/CLAUDE.md`) so the identity + memory protocol follows you into every repo. Set the env var `MEMORY_DIR` to this repo's `memory/` so the brain scripts always find home.
