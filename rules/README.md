# rules/

Always-on working rules the assistant follows across every project — coding
style, security, testing, subagent usage, and performance/model selection.

These are opinionated defaults, not laws. Edit any file to match your own
standards, delete what doesn't apply to you, and add new rule files for
anything else you want enforced consistently (e.g. `accessibility.md`,
`documentation.md`).

Keep each file short and concrete — it gets loaded into the assistant's
context on every relevant task, so terse and specific beats long and vague.
